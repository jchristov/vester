run

The entry point to the whole system.

Parameters

  • config Object The Vester configuration.
    • config.journal string
    • config.backtesting boolean
    • config.slackUrl string
    • config.initialStates Object
    • config.feeds Object
    • config.backtest Object
      • config.backtest.timestamp string
    • config.commission (function | number)
    • config.guard Object
      • config.guard.shorting boolean
      • config.guard.margin boolean
      • config.guard.restricted Array

Examples

import run from 'vester'

function strategy({ order }, action) {
  order({
    identifier: 'AAPL',
    quantity: 100,
    price: 150
  })
}

run({
  strategy,
  backtesting: false
})

Returns Promise

strategy

The strategy function is defined by the user (you), and it is called every time a new event occurs.

Parameters

  • context Object
    • context.state function The state of your strategy.
    • context.metrics function Some standard metrics for your strategy. Note that calls to this function is very expensive, so use with caution.
    • context.order function Place an order.
    • context.cancel function Cancel an order.
